🛠️ Tech Stack & Architecture
Frontend (SPA Workspace)
Vite & React 18: Ultra-fast, state-driven rendering canvas.
Tailwind CSS: Modular styling variables.
Framer Motion: High-fidelity fluid interface transitions.
React Markdown & Highlight.js: High-precision Markdown rendering with syntax highlighting.
Backend (Orchestration Engine)
Node.js & Express: Clean, decoupled Monolithic REST API.
Sequelize ORM & SQLite: Stable, self-healing relational local storage.
JWT & Bcrypt: Secure, token-based local authentication.
Hugging Face / OpenAI Router: Secure asynchronous API integration with automatic model provider load balancing.

🚀 Local Installation & Quick Start
Prerequisites
Ensure you have Node.js (v18+) installed.
1️⃣ Clone and Initialize
Clone the repository, then install dependencies for all layers (Root, Backend, and Frontend) simultaneously:
code
Bash
# Clone the repository
git clone https://github.com/YOUR_USERNAME/flutter-hub.git
cd flutter-hub

# Install all packages across the Monorepo (Cross-platform prefix command)
npm run install:all
2️⃣ Configure Credentials
Go to the backend folder.
Duplicate the .env.example file and rename it to .env:
code
Bash
cp backend/.env.example backend/.env
Open backend/.env and insert your credentials:
Provide your HF_TOKEN (Hugging Face Token).
Set a strong JWT_SECRET for local password hashing.
3️⃣ Fire Up the Development Server
Execute the synchronized development script from the root directory:
code
Bash
npm run dev
This command fires up the Express API Server (Port 5000) and the Vite React Dev Client (Port 5173) in parallel with real-time colored console logging.
🔒 Security & Data Privacy
API Protection: Local API keys are strictly stored inside backend/.env and are entirely excluded from version control via .gitignore.
Database Privacy: All personal chats, credentials, and custom session configurations are stored completely offline within the local database.sqlite file.
